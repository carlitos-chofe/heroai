## Why

The current backend relies on Google's Vertex AI for story and image generation using the `google-genai` SDK with `vertexai=True`. However, Vertex AI strictly requires Google Cloud Application Default Credentials (ADC) to authenticate and does not support simple API keys. Because the Celery `worker` and `api` services run inside isolated Docker containers, they cannot access the ADC credentials generated by `gcloud init` on the host LXC machine, resulting in a `"Your default credentials were not found"` error.

## What Changes

- Map the host machine's Google Cloud credentials directory (`~/.config/gcloud/`) to the Docker containers (`api` and `worker`) as a read-only volume.
- Set the `GOOGLE_APPLICATION_CREDENTIALS` environment variable in the `docker-compose.yml` file to explicitly point the SDK to the mounted ADC JSON file inside the containers.

## Capabilities

### New Capabilities
- `vertex-ai-auth`: Authentication mechanism for Vertex AI inside Docker containers.

### Modified Capabilities
- (None - this is an infrastructure/configuration fix, no functional requirements are changing)

## Impact

- `docker-compose.yml`: Will be updated with new volume mounts and environment variables for the `worker` and `api` services.
- **Security**: The containers will have read-only access to the host's Google Cloud credentials. This is suitable for local development.
- **Developer Experience**: Developers will be able to seamlessly use Vertex AI models locally by simply running `gcloud init` or `gcloud auth application-default login` on their host machine.
